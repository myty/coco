## Plan

### Summary

After the local Anthropic-compatible proxy starts, Claudio must:

1. Check whether `claude` (Claude Code CLI) is installed
2. If not, print predictable installation instructions and exit
3. If yes, spawn it as a subprocess with inherited stdio and the proxy's env
   vars
4. Wait for it to exit, then stop the proxy and propagate the exit code

No SDK or external tooling — pure `Deno.Command` + process lifecycle management.

### Design Notes

#### Binary Detection

`findClaudeBinary()` tries in order:

1. `claude` on `PATH` via `Deno.Command("which"/"where")`
2. Known npm global install paths (`~/.npm-global/bin/claude`, etc.)
3. Returns `null` if not found

#### Environment Variables Set on Spawn

| Variable             | Value                                                   |
| -------------------- | ------------------------------------------------------- |
| `ANTHROPIC_BASE_URL` | `http://127.0.0.1:<port>`                               |
| `ANTHROPIC_API_KEY`  | `"claudio"` (any non-empty value satisfies Claude Code) |

All existing env vars are inherited; only these two are added/overridden.

#### Subprocess Lifecycle

```
main()
  └─ ensureAuthenticated()
  └─ startServer()  →  returns { port, stop() }
  └─ findClaudeBinary()
       ├─ null → printInstallInstructions() → await stop() → Deno.exit(1)
       └─ path → launchClaudeCode(path, port, forwardedArgs)
                    └─ Deno.Command(...).spawn()  [inherited stdio]
                    └─ await status
                    └─ return status.code ?? 1
  └─ (try/finally) await stop()    ← proxy shutdown always runs
  └─ Deno.exit(exitCode)
```

#### Forwarded Args

All `Deno.args` entries that are not Claudio flags (`--help`, `--version`,
`--server`) are forwarded to `claude` verbatim.

#### Installation Instructions (predictable, minimal)

```
Claude Code is not installed.

To install:
  npm install -g @anthropic-ai/claude-code

Or download directly:
  https://claude.ai/download
```
