Why Ardo

You already have GitHub Copilot. You shouldn't need another subscription to use your favorite agents.

One login, really

Authenticate once with GitHub Copilot. Every agent you add after that routes through the same subscription — no new API keys, no new billing.

Stays on your machine

Ardo binds to localhost only. It proxies requests to GitHub Copilot and never becomes a hosted service or remote control plane.

Easy to undo

Agent config is reversible by design. Run ardo unconfigure and your agents go back to exactly where they were.

What It Does

Translate the compatibility layer, not your workflow.

Familiar protocol surface

Ardo exposes Anthropic compatible and OpenAI compatible local endpoints so supported tools can point at one place.

Background service controls

Start, stop, restart, inspect status, and optionally install Ardo as an OS login service.

Agent-aware configuration

Configure and unconfigure supported agents from the CLI while keeping configuration state explicit.

Current built-in agent support

  • Claude Code
  • Cline
  • Codex

Ardo is intentionally narrow: it stays focused on being a calm, local gateway rather than becoming a chat client or an agent runtime of its own.

Install

The recommended path should be obvious.

Most users should install with npm. Deno, direct binary, and source-based setup remain available when they fit your environment.

Recommended

npm

Use this if you want the shortest path on a machine with Node.js 18+.

npm install -g ardo
  1. Install Ardo globally.
  2. Run ardo start and complete GitHub authentication.
  3. Run ardo configure claude-code or another supported agent.

Deno / JSR

Useful when Deno is already part of your toolchain.

deno install -A -g jsr:@ardo-org/ardo

Direct binary

Download a platform build from GitHub Releases when you do not want a runtime dependency.

Open releases

From source

Good for contributors or if you want to inspect the code before installing.

git clone https://github.com/ardo-org/ardo.git cd ardo deno task install

How It Works

A local request path with a narrow, visible role.

Coding agent

Claude Code, Cline, or Codex sends a request to a local endpoint.

→

Ardo

Ardo runs on 127.0.0.1, exposes compatible endpoints, and translates request and response shapes.

  • POST /v1/messages
  • POST /v1/chat/completions
  • GET /v1/models
  • GET /health
→

GitHub Copilot

Ardo authenticates with GitHub and forwards the translated request to Copilot-backed models.

Trust boundary: Ardo is a local gateway, not a hosted relay. It binds to localhost and keeps configuration changes reversible.

FAQ

Common questions before you decide to install.

Do I need a GitHub Copilot subscription?

Yes. Ardo routes requests through GitHub Copilot, so you need an active Copilot-backed account.

Does Ardo expose an API on my network?

No. The server configuration binds to 127.0.0.1, which keeps the service local to your machine.

Can I undo a configuration change?

Yes. Ardo supports unconfiguring supported agents and is designed around reversible configuration management.

Which agents are supported right now?

The current built-in registry includes Claude Code, Cline, and Codex.

Do I need Deno to use Ardo?

No. npm and direct binary installs do not require Deno. Deno is only needed for the JSR and source-based paths.
